# Sound Quest OpenAI TTS server

This folder runs `index.html` through a small local server.
When opened from this server, the page asks `/api/tts?word=bet` for an mp3 generated by OpenAI TTS.

## Setup

1. Copy `.env.example` to `.env`.
2. Put your OpenAI API key in `.env`.
3. Run with PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\outputs\sound-quest\server.ps1
```

Or run with Python if you have Python installed:

```powershell
python .\outputs\sound-quest\server.py
```

If your Windows uses the Python launcher:

```powershell
py .\outputs\sound-quest\server.py
```

4. Open:

```text
http://127.0.0.1:8787
```

Generated audio files are cached in `audio-cache/`, so the same word does not need to call the API every time.

## Vercel

This folder can be deployed to Vercel.

Set these Environment Variables in Vercel:

```text
OPENAI_API_KEY=your_api_key
OPENAI_TTS_MODEL=gpt-4o-mini-tts
OPENAI_TTS_VOICE=alloy
```

If you do not set `OPENAI_API_KEY`, words fall back to browser speech.
